Ejercicio: Implementación de una Conexión Única a la Base de Datos

Descripción:

Vas a implementar un sistema de conexión a una base de datos que solo permita una única instancia activa en toda la aplicación. El objetivo es garantizar que no se creen múltiples conexiones innecesarias a la base de datos, lo que puede causar problemas de rendimiento y gestión de recursos.

Requerimientos:

  1. Clase DatabaseConnection:
    • Implementa la clase DatabaseConnection utilizando el patrón Singleton.
    • La clase debe tener un método connect() que simule la conexión a la base de datos. Este método puede simplemente retornar un mensaje que indique que la conexión ha sido establecida.
    • Implementa un método disconnect() que simule la desconexión de la base de datos.
  2. Validación de Instancia Única:
    • Asegúrate de que cada vez que se intente crear una nueva instancia de DatabaseConnection, se devuelva la misma instancia que la anterior.
  3. Prueba del Singleton:
    • Crea dos instancias de DatabaseConnection.
    • Conecta a la base de datos usando ambas instancias.
    • Verifica que ambas instancias son idénticas y que la conexión se establece solo una vez.
